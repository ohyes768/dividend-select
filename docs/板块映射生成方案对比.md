# 个股板块映射生成方案对比

## 问题背景
需要生成一个 CSV 文件，包含所有 A 股的个股板块映射关系（股票代码 → 概念板块 + 行业板块）

---

## 方案对比

### 方案一：当前方案（fetch_board_mapping.py）

**原理**：
- 先批量获取股票基本信息
- 然后对每只股票调用 `ef.stock.get_belong_board(code)` 获取板块

**优点**：
- 逻辑简单直接
- 可以精确获取每只股票的板块信息

**缺点**：
- ❌ 需要调用 ~5500 次 API（5489 只股票）
- ❌ 内存消耗大（一次性创建所有任务）
- ❌ 容易被限流/超时
- ❌ 无断点续传，中断需要重跑

**预计耗时**：15-30 分钟（实际测试中因内存不足失败）

---

### 方案二：批量 API 方案（fetch_board_mapping_v2.py）

**原理**：
- 使用 `ef.stock.get_all_base_info()` 一次性获取所有板块的股票列表
- 反向构建股票 → 板块的映射

**优点**：
- ✅ 只需几次 API 调用即可完成
- ✅ 内存消耗小
- ✅ 速度快

**缺点**：
- ⚠️ 依赖 efinance 的批量 API 返回格式
- ⚠️ 需要确认 API 是否返回所需数据

**预计耗时**：2-5 分钟

---

### 方案三：混合方案（推荐，最稳定）

**原理**：
1. 从 akshare/efinance 获取所有股票列表
2. 使用 efinance 获取所有板块列表及其成分股
3. 反向构建映射

**优点**：
- ✅ API 调用量最少
- ✅ 逻辑清晰，易于维护
- ✅ 可加入断点续传

**缺点**：
- 需要确认数据源的 API 格式

**预计耗时**：1-3 分钟

---

## 推荐实现（方案三简化版）

```python
"""
核心思路：
1. 获取所有板块列表
2. 获取每个板块的成分股
3. 反向构建 股票→板块 映射
"""

import efinance as ef
import pandas as pd
from collections import defaultdict

def get_board_mapping():
    # 1. 获取所有板块
    boards = ef.stock.get_all_base_info()  # 返回格式需要确认

    # 2. 构建映射
    stock_to_boards = defaultdict(set)
    for _, row in boards.iterrows():
        stock_code = row['股票代码']
        board_name = row['板块名称']
        stock_to_boards[stock_code].add(board_name)

    # 3. 导出
    df = pd.DataFrame([
        {
            '股票代码': code,
            '概念板块': ';'.join(sorted(boards))
        }
        for code, boards in stock_to_boards.items()
    ])
    df.to_csv('data/个股板块映射.csv', index=False, encoding='utf-8-sig')
```

### 2. 断点续传机制
```python
import json
import os

# 保存进度
def save_progress(data, file='data/progress.json'):
    with open(file, 'w') as f:
        json.dump(data, f)

# 加载进度
def load_progress(file='data/progress.json'):
```

## API 测试结果

| API | 状态 | 说明 |
|-----|------|------|
| `ef.stock.get_all_base_info()` | ❌ 不存在 | efinance 没有此方法 |
| `ef.stock.get_sectors()` | ❌ 不存在 | efinance 没有此方法 |
| `ef.stock.get_members(code)` | ⚠️ 超时 | API 可用但网络慢 |
| `ef.stock.get_belong_board(code)` | ⚠️ 可用但慢 | 单只股票，网络不稳定 |

**结论**：efinance 没有批量获取板块-股票关系的 API，必须逐个股票调用 `get_belong_board()`

## 最终推荐方案：稳健版（fetch_board_mapping_v3.py）

### 核心优化

1. **串行处理**（不用多线程）
   - 降低内存消耗
   - 避免 API 限流

2. **断点续传**
   - 每 50 只股票保存一次进度
   - 中断后可继续，不用重跑

3. **自动重试**
   - 失败自动重试 3 次
   - 指数退避避免频繁请求

### 预计耗时

- 首次运行：**5-10 分钟**（5489 只股票，约 0.05-0.1 秒/只）
- 断点续传：只处理剩余部分

### 使用方法

```bash
# 首次运行
python fetch_board_mapping_v3.py

# 中断后继续运行（自动从断点继续）
python fetch_board_mapping_v3.py
```
