# 股息率表格技术指标扩展 - 技术规范

## 1. 概述

### 1.1 背景

现有股息率展示页面已包含股票基本信息、股息率数据、申万行业等信息。后端新增了 PE（市盈率）、PB（市净率）、M120（120日均线）、最新收盘价及偏离度等技术指标数据。需要在前端表格中新增展示这些指标。

### 1.2 目标

**核心目标：**
- 在现有表格中新增技术指标列
- 展示 PE、PB、M120、收盘价、偏离度数据
- 保持表格可读性和性能

**附加目标：**
- 支持数据并行加载优化
- 偏离度颜色编码区分正负
- 支持单只股票实时股价刷新
- 本地缓存刷新后的偏离度数据

**非目标：**
- 技术指标列不支持排序
- 不提供技术指标历史趋势图
- 刷新后数据不持久化到后端（仅本地缓存）

## 2. 功能需求

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| PE/PB 列展示 | 合并展示市盈率和市净率 | P0 |
| M120 列展示 | 合并展示 M120、收盘价、偏离度 | P0 |
| 并行数据加载 | 同时加载股息率、PE、M120 数据 | P0 |
| 偏离度颜色编码 | 红涨绿跌区分正负偏离 | P0 |
| 实时股价刷新 | 点击按钮刷新单只股票的实时股价和偏离度 | P0 |
| 本地缓存 | 使用 localStorage 缓存刷新后的偏离度 | P0 |

### 2.2 用户故事

**场景1：查看估值指标**
用户在表格中可以直接看到每只股票的 PE 和 PB，快速判断股票估值水平。

**场景2：判断股价趋势**
用户通过 M120 和偏离度数据，了解当前股价相对 120 日均线的位置，判断买入/卖出时机。

**场景3：综合分析**
用户结合股息率、PE/PB、M120 偏离度，筛选出股息率高、估值合理、股价处于合理区间的股票。

**场景4：实时刷新偏离度**
用户在交易时间想要查看某只股票的当前偏离度，点击该行的"刷新"按钮，系统获取最新收盘价并计算新的偏离度，更新表格显示并缓存到本地。

└─────────────────┘
         │

│                                      └────────────┘
└─────────────────┘

**数据合并策略：**
- 使用股票代码作为 key 合并三个数据源
- 缺失数据显示为 "-"
- 使用 `Promise.all` 并行加载

**本地缓存策略：**
- 缓存键：`dividend-deviation-{code}`
- 缓存内容：`{ close, deviation, timestamp }`
- 缓存时长：1 天（86400 秒）
- 读取顺序：本地缓存 > 远程接口

### 3.4 方案权衡记录

**并行加载 vs 串行加载**
- 考虑：三个接口独立，无依赖关系
- 选择：并行加载
- 理由：减少总加载时间，提升用户体验

**合并列 vs 独立列**
- 考虑：表格列数过多影响可读性
- 选择：PE/PB 合并，M120/收盘/偏离度合并
- 理由：保持表格宽度可控，数据紧凑展示

**排序支持**
- 考虑：技术指标主要用于参考
- 选择：不提供排序
- 理由：简化实现，保持按股息率排序的核心逻辑

**刷新范围**
- 考虑：批量刷新可能超出接口限制
- 选择：单只股票刷新
- 理由：避免同时发起过多请求，接口实现简单

**缓存持久化**
- 考虑：刷新数据无需保存到后端，降低复杂度
- 选择：localStorage 本地缓存
- 理由：实现简单，1天缓存足够满足需求

## 4. 数据设计

### 4.1 数据结构

#### 技术指标数据模型

```typescript
interface PEData {
  code: string;
  name: string;
  pe: number | null;        // 市盈率
  pb: number | null;        // 市净率
  market_cap: number | null;        // 总市值（万元）
  circulation_market_cap: number | null;  // 流通市值（万元）
}

interface M120Data {
  code: string;
  name: string;
  avg_yield_3y: number | null;
  m120: number | null;       // 120日均线
  close: number | null;      // 最新收盘价
  deviation: number | null;  // 偏离度(%)
}

interface TechnicalIndicators {
  pe: number | null;
  pb: number | null;
  m120: number | null;
  close: number | null;
  deviation: number | null;
}
```

#### 合并后的股票数据

```typescript
interface DividendStockWithTechnical extends DividendStock {
  technical?: TechnicalIndicators;
}
```

#### 实时股价刷新数据模型

```typescript
// 后端接口请求体
interface RealtimePriceRequest {
  code: string;      // 股票代码
  m120: number;      // 120日均线值
}

// 后端接口响应体
interface RealtimePriceResponse {
  code: string;
  close: number | null;   // 最新收盘价
}

// 前端缓存数据
interface DeviationCache {
  close: number;
  deviation: number;
  timestamp: number;  // 时间戳
}

// 前端刷新状态
interface RefreshState {
  code: string;
  loading: boolean;
  error: string | null;
}
```

### 4.2 状态管理

**状态层级：**

DividendPage

  ├─ cache: Map<string, DeviationCache>         // 本地缓存

### 5.2 展示规范

**PE/PB 列：**
- 格式：`PE: {数值} / PB: {数值}`
- 数值格式：保留 2 位小数
- 无数据时显示：`-`
- 列宽建议：`w-36`

**M120/收盘/偏离度 列：**
- 格式：
  ```
  M120: {数值} / 收盘: {数值}
  偏离度: {数值}%
  ```
- 数值格式：保留 2 位小数
- 偏离度带 + 符号（正偏离）或 - 符号（负偏离）
- 无数据时显示：`-`
- 列宽建议：`w-48`

**偏离度颜色编码：**
- 正偏离（高于 M120）：红色 `text-red-500`
- 负偏离（低于 M120）：绿色 `text-green-500`
- 无数据：灰色 `text-gray-400`

**刷新按钮设计：**
- 图标：🔄 或刷新图标
- 文字："刷新"
- 位置：操作列底部，与其他按钮并列
- 状态：
  - 正常状态：灰色/蓝色，可点击
  - 加载状态：旋转动画，禁用点击
  - 错误状态：红色，可重试
- 尺寸：与其他操作按钮保持一致
- Hover 效果：颜色加深

### 5.3 交互设计

**加载状态：**
- 初始显示骨架屏或加载指示器
- 数据加载完成后更新表格

**刷新按钮交互：**
1. 用户点击刷新按钮
2. 按钮进入加载状态（旋转动画）
3. 发起实时股价请求
4. 计算新偏离度
5. 更新 M120/收盘/偏离度列显示
6. 保存到 localStorage 缓存
7. 按钮恢复正常状态

**刷新流程：**
```
点击刷新 → 检查缓存 → 缓存有效？ → 是 → 使用缓存数据
                                    ↓ 否
                              发起实时股价请求
                                    ↓
                              计算新偏离度
                                    ↓
                              更新表格显示
                                    ↓
                              保存到 localStorage
```

**错误处理：**
- 部分数据加载失败：显示已加载数据，缺失数据显示 "-"
- 全部加载失败：显示错误提示和重试按钮
- 刷新失败：显示 Toast 错误提示，保持原有数据显示

**悬停效果：**
- 技术指标列单元格无特殊悬停效果
- 保持与现有表格一致的行悬停效果
- 刷新按钮悬停时颜色加深

## 7. 边缘情况与错误处理

### 7.1 异常场景

**网络异常：**
- 部分接口失败：其他接口正常返回，缺失数据显示 "-"
- 全部接口失败：显示错误提示

**数据异常：**
- PE/PB 数据缺失：显示 "-"
- M120 数据缺失：显示 "-"
- 偏离度为 null：显示 "-"

### 7.2 边界条件

**极端值：**
- PE > 999：正常显示，不截断
- 偏离度 > 100%：正常显示
- 偏离度 < -100%：正常显示

**数据量为 0：**
- 技术指标数据为空：表格正常显示，技术指标列全部显示 "-"

### 7.3 降级策略

- 技术指标接口失败：正常显示股息率数据，技术指标列显示 "-"
- 数据合并失败：降级为只显示股息率数据
- 刷新接口失败：显示 Toast 错误提示，保持原有数据
- 缓存读取失败：降级为调用远程接口
