# 股票板块映射更新程序 - 技术规范文档

## 1. 概述

### 1.1 背景
当前 `个股板块映射.csv` 文件包含了股票的概念板块和行业板块信息，但数据可能不是最新的。需要开发一个程序，从 `红利指数持仓汇总.csv` 读取股票列表，通过 efinance API 批量查询最新的板块信息，并生成新的映射文件。

> **注**: 本技术规范已实现为 `src/data/board_fetcher.py` 中的 `BoardMappingFetcher` 类。

### 1.2 目标
- **核心目标**：基于红利指数持仓汇总.csv，获取每只股票最新的概念板块和行业板块信息
- **附加目标**：确保股票代码格式正确，自动处理交易所后缀转换
- **非目标**：不保留原有数据，完全重新生成映射文件

## 2. 功能需求

### 2.1 核心功能
1. 读取 `红利指数持仓汇总.csv` 获取股票列表
2. 自动转换股票代码格式（根据交易所字段添加 .SH/.SZ 后缀）
3. 使用 akshare API 查询每只股票的概念板块和行业板块
4. 将结果保存为 `个股板块映射.csv`

### 2.2 用户故事
作为数据分析师，我需要一个工具能够自动从红利指数持仓中提取股票代码，查询最新的板块分类信息，并保存为CSV文件，以便后续进行板块分析。

### 2.3 数据流向
```
红利指数持仓汇总.csv → 股票代码提取 → 格式转换 → akshare API查询 → 数据清洗 → 个股板块映射.csv
```

## 3. 技术决策

### 3.1 技术栈
| 组件 | 选择 | 理由 |
|------|------|------|
| 编程语言 | Python 3.x | 数据处理脚本的标准选择 |
| 数据源 | efinance + akshare | 与现有代码风格一致，efinance获取板块信息 |
| CSV处理 | pandas | 高效处理结构化数据 |
| 编码格式 | UTF-8 with BOM (utf-8-sig) | 与原文件保持一致，Excel兼容 |

### 3.2 架构设计

### 3.3 方案权衡记录

| 决策点 | 备选方案 | 最终选择 | 理由 |
|--------|----------|----------|------|
| 查询策略 | 串行查询 / 并发查询 | 串行查询 | API稳定性优先，避免被限流 |
| 错误处理 | 立即停止 / 跳过失败 / 重试机制 | 跳过失败 | 单个股票失败不影响整体进度 |
| 空值处理 | 保留空字段 / 标记为无 / 跳过记录 | 标记为"无" | 数据完整性，便于后续识别 |
| 代码组织 | 单文件脚本 / 模块化设计 | 模块化设计 | 可维护性和可扩展性 |
| 股票代码格式 | 6位代码 / 带后缀代码 | 6位代码 | efinance 支持6位代码，无需转换 |

### 3.4 潜在风险与缓解措施

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| efinance API限流 | 中 | 串行查询，每次请求间隔1秒 |
| API数据结构变更 | 低 | 使用稳定的API接口 |
| 股票代码不存在 | 低 | 跳过失败，记录股票代码 |
| 网络不稳定 | 中 | 添加超时设置，失败自动跳过 |

## 4. 数据设计

### 4.1 数据结构

#### 输入数据结构（红利指数持仓汇总.csv）
```python
class DividendStock:
    exchange: str      # 交易所（深市主板/沪市主板/创业板）
    stock_code: str    # 股票代码（6位数字）
    stock_name: str    # 股票名称
    source_index: str  # 来源指数
    # ... 其他字段
```

#### 输出数据结构（个股板块映射.csv）
```python
class StockSectorMapping:
    stock_code: str      # 股票代码（6位，如000090）
    stock_name: str      # 股票简称
    concept_sector: str  # 概念板块（分号分隔）
    industry_sector: str # 行业板块（分号分隔）
```

### 4.2 API 说明
```python
# efinance API 调用方式
ef.stock.get_base_info([code])      # 获取基本信息（含所属行业）

```

## 7. 边缘情况与错误处理

### 7.1 异常场景
| 场景 | 处理策略 |
|------|----------|
| 网络超时 | 跳过该股票，记录到失败列表 |
| API返回错误 | 跳过该股票，记录到失败列表 |
| 股票代码不存在 | 跳过该股票，记录到失败列表 |
| 交易所字段异常 | 记录警告，使用默认后缀.SH |

### 7.2 数据边界条件
| 条件 | 处理方式 |
|------|----------|
| 概念板块为空字符串 | 标记为"无" |
| 行业板块为空字符串 | 标记为"无" |
| 股票名称包含特殊字符 | 保留原始字符 |

### 7.3 降级策略
- 如果akshare API不可用，提示用户检查网络或API状态
- 如果文件写入失败，提示检查文件权限
